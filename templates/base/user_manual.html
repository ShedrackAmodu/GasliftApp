1 Introduction

The Gas Lift Opportunity Automation System is a web-based application designed to help petroleum engineers and production technologists identify wells that are good candidates for gas lift installation. By analyzing historical well test data, the system applies statistical trend analysis and multi-criteria scoring to rank wells based on their potential for gas lift optimization.

Key Features
Data Upload

Upload CSV or Excel well test data with automatic column detection

Column Mapping

Map your file columns to required system fields

Trend Analysis

Statistical trend detection using Kendall's tau and Sen's slope

Candidate Ranking

Multi-parameter weighted scoring with gas allocation optimization

Liquid Loading Diagnostics

PVT-based critical velocity analysis for liquid loading detection

Export Options

Export results to Excel, CSV, or PDF executive summary

2 Getting Started

System Requirements
  • A modern web browser (Chrome 90+, Firefox 88+, Edge 90+, Safari 14+)
  • An active internet connection
  • A registered user account
Creating an Account
  1. Navigate to the login page and click "Sign Up".
  2. Enter your username, email address, and password.
  3. Click "Create Account" to register.
  4. You will be automatically logged in and redirected to the Dashboard.

After logging in, you will see the Dashboard with four summary statistics:

4 Step 2: Column Mapping

The column mapping page allows you to match the columns in your uploaded file to the system's required fields. This ensures the application correctly reads and interprets your data.

How to Map Columns
  1. For each required field on the left, select the matching column from your file on the right.
  2. If a column name in your file already matches a required field, it will be pre-selected automatically.
  3. Click "Validate & Save Mapping" to confirm.
  4. The system will validate that all required fields are mapped before proceeding.
Note: The "Well" and "Date" fields are mandatory. Without them, the analysis cannot proceed.

6 Step 4: Adjust Weights & Parameters

This page lets you configure the scoring parameters that determine how wells are ranked. Each parameter can be weighted according to its importance in your gas lift candidate selection criteria.

Weight Parameters
ParameterDefault WeightDescription
BS&W (%)100Weight for water cut increase trends
Oil Rate (bopd)100Weight for oil rate decline trends
GLR (scf/bbl)100Weight for gas-liquid ratio decline
Tubing Pressure (psi)50Weight for tubing pressure decline
Economic Oil Limit50 bopdOil rate threshold for economic limit calculation
Advanced Options
  • Gas Constraint (MMscf/d) — Available lift gas volume for knapsack optimization. If set, the system will allocate gas optimally across candidates.
  • Base Choke Size — Choke size for normalization across all wells
  • Outlier Detection — Method (IQR, Z-score) and threshold for outlier removal
Optional: Configure PVT Properties

For liquid loading diagnostics, you can configure fluid properties:

  • API Gravity — Oil API gravity (default: 35)
  • Gas Specific Gravity — Relative to air (default: 0.65)
  • Water Salinity — In ppm (default: 20,000)
  • Reservoir Temperature — In °F (default: 180)
  • Tubing ID — Tubing inside diameter in inches (default: 2.875)
Optional: Upload Completion Data

For feasibility checking, you can upload completion data including mandrel depths, packer depth, and available compression pressure.

Tip: Higher weights (up to 100) give more influence to a parameter in the final candidate score. Adjust based on your field's specific gas lift candidate criteria.

7 Step 5: Well Trends

The Well Trends page provides an interactive view of individual well performance over time. You can inspect each well's data trends before running the full analysis.

Features
  • Well Selector — Choose a well from the dropdown to view its data
  • Interactive Charts — Four synchronized charts showing:
    • BS&W (%) over time
    • Net Oil (bopd) over time
    • Form.GLR (scf/bbl) over time
    • Tubing Pressure (psi) over time
  • Data Point Details — Hover over any point to see exact values
  • Outlier Markers — Rejected data points are visually marked
  • Data Quality Score — Displays the percentage of valid data
Running the Analysis

Once you've reviewed the trends, click "Run Analysis" to start the automated candidate scoring. The system will:

  1. Apply Sen's slope estimation for trend direction and magnitude
  2. Calculate Kendall's tau-b correlation for statistical significance
  3. Compute weighted candidate scores based on your configured parameters
  4. Rank wells from best to least candidate potential
  5. Optionally run gas knapsack optimization if gas constraint is set
Note: Analysis time depends on the number of wells and data points. Progress is shown via the loading overlay.

8 Step 6-7: View Results

The Results page displays the ranked list of candidate wells based on the analysis.

Results Table

The main table shows each well with:

  • Well ID — Well name/identifier
  • Candidate Score — Composite score (higher is better)
  • BSW Flag — Red if BS&W trend is increasing (bad)
  • Oil Rate Flag — Red if oil rate is declining (bad = good gas lift candidate)
  • GLR Flag — Red if GLR is declining
  • Trend Details — Direction and magnitude for each parameter
  • Liquid Loading — Flag if liquid loading is detected
  • Completion Feasibility — Green/yellow/red status
  • Summary Comment — Auto-generated description of the well's situation
Filtering & Sorting
  • Use the filter buttons (BSW Flag, Oil Flag, GLR Flag) to focus on specific well categories
  • Use the search bar to find wells by name
  • Filter by production method
Comparing Analyses

The Compare Analyses feature allows you to select two analysis sessions and view side-by-side ranking changes, including:

  • Rank improvements or deteriorations
  • Score changes
  • New wells appearing in the results

10 Advanced Features

Sensitivity Analysis (Monte Carlo)

The sensitivity analysis uses Monte Carlo simulation to assess the robustness of well rankings under uncertainty. It runs 1,000 iterations with random perturbations to input parameters and measures rank stability.

  • Rank Variance — Lower variance means more stable ranking
  • Confidence Classification — Wells are classified as High, Medium, or Low confidence
  • Requires at least 2 wells for meaningful results
Liquid Loading Diagnostics

When PVT properties are configured, the system calculates:

  • Critical Gas Velocity — Minimum velocity needed to lift liquids (ft/s)
  • Actual Gas Velocity — Current estimated gas velocity
  • Liquid Loading Flag — Set when actual velocity falls below critical
Gas Allocation Optimization (Knapsack)

When a gas constraint is specified, the system applies a greedy knapsack algorithm to allocate available lift gas across candidates to maximize incremental oil gain. Wells are ranked by efficiency (candidate score per unit gas).

Economic Limit Forecasting

Using the economic oil rate limit and Sen's slope, the system estimates:

  • Days to Economic Limit — How long before the well becomes uneconomic
  • Projected Oil Rate (6 months) — Forecasted rate based on current decline
Completion Feasibility

With completion data uploaded, the system checks whether available compression pressure is sufficient for gas lift at the deepest mandrel, considering hydrostatic and friction losses.

11 FAQ & Troubleshooting

Scores are calculated using a weighted combination of four trend parameters: BS&W, Oil Rate, GLR, and Tubing Pressure. Each parameter's trend direction, statistical significance (p-value), and magnitude are combined with user-configurable weights. Wells showing declining oil rate, increasing BSW, or declining GLR receive higher scores as they represent better gas lift candidates.

The system supports Excel (.xlsx) and CSV (.csv) files. Maximum file size is 100 MB. For Excel files, only the first sheet is read. Ensure your data is in tabular format with column headers in the first row.

Common issues:
  • File upload fails: Check file size (max 100 MB) and format (.xlsx or .csv)
  • No wells analyzed: Ensure all required columns are mapped and wells have ≥5 data points
  • Analysis fails: Check for non-numeric values in numeric columns (BS&W, Oil, GLR, Pressure)
  • Charts not loading: Ensure your browser supports JavaScript and Chart.js
